根因聚类

聚类 A范围 / 覆盖度
影响 差异 1、2、4。根因：Agent 的任务心智模型停留在"颜色 + 字体"与"一个示例表面"。修复：把所有标尺块（spacing / rounded）枚举齐全，并把组件清单作为显式检查清单逐项遍历。仅需 prompt 改动，不改 runtime 代码。
聚类 B变量类型与绑定架构
影响 差异 3、7。根因：Figma 无 typography 复合类型，Agent 选择了不可绑定的 JSON-in-STRING 表示。修复：以专用工具（import_design_md_typography）显式承载分解策略，直接产出原子变量和配套绑定。差异 7 随 差异 4 一同消解。
聚类 C节点生命周期与状态一致性
影响 差异 5、6。根因：bind_variable 按名字解析，静默撞上遗留集合；Agent 引用的节点 ID 已不在当前 turn 的工作记忆中。修复：runtime 护栏——集合作用域绑定 + turn 边界同步 / jsx 返回完整子树 idMap。
