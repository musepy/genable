# /src

> Core plugin source - UI Thread (React) + Sandbox Thread (Figma) + DSL processing.
> 双线程架构的核心实现，所有业务逻辑都在这里。
> 子目录：`/knowledge` (组件知识库), `/services` (API 集成), `/playground` (实验功能)

---

## 文件清单 (File Inventory)

| 文件 | 地位 | 功能 |
|------|------|------|
| `main.ts` | **核心** | Sandbox 线程入口，DSL → Figma 节点渲染 |
| `ui.tsx` | **核心** | UI 线程入口，React 界面 + API 调用编排 |
| `schema.ts` | **契约** | Zod DSL Schema，LLM 输出验证 |
| `contextBuilder.ts` | **编排** | LLM 上下文构建，System Prompt 生成 |
| `types.ts` | 基础 | TypeScript 类型定义，消息协议 |
| `constants.ts` | 基础 | 常量定义 (模板、加载步骤) |
| `tokenMapper.ts` | 功能 | 原始值 → Figma Variables 映射 |
| `patternRecognition.ts` | 功能 | 设计模式识别与聚类 |
| `layerFilter.ts` | 功能 | 图层质量过滤 |
| `styleMemory.ts` | 功能 | 用户风格偏好记忆 |
| `designExamples.ts` | 数据 | Few-shot 示例库 |
| `presets.ts` | 数据 | 预设检测规则 |
| `styles.css` | 样式 | UI 样式表 |
| `styles.css.d.ts` | 类型 | CSS 模块类型定义 |

---

## 数据流 (Data Flow)

```
User Prompt (ui.tsx)
       ↓
Context Builder (contextBuilder.ts)
       ↓ + Variables + Patterns + Knowledge
Gemini API (services/gemini.ts)
       ↓ JSON DSL
Schema Validation (schema.ts)
       ↓
postMessage → Sandbox Thread
       ↓
Renderer (main.ts → renderLayer)
       ↓
Figma SceneNodes
```
