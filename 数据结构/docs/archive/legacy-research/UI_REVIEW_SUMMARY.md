## 🔴 严重问题详解

### 1. Token 占位符 (css.ts 行 963, 988-1007)

**问题**:
```css
--panel-default: {Panel.translucent};  ❌
--tokens-colors-accent-contrast: {Tokens.Colors.white-contrast};  ❌
--tokens-space-button-height-1: {Spacing.5};  ❌
/* 共 6 个类似错误 */
```

**根本原因**: Figma 设计系统导出的占位符未经过转换工具处理

**用户影响**: 
- 部分样式无效，背景/边框/间距无法显示
- 布局错位，色彩不符设计

**修复难度**: ⭐ 低 (30 分钟)

---

### 2. Color Token 缺失 (css.ts)

**问题**:
```typescript
// colors.ts 声明（有这些引用）
success: 'var(--success-9)',
error: 'var(--error-9)',
warning: 'var(--warning-9)',

// css.ts 定义（缺少这些变量）
/* ❌ --success-1 到 --success-12 不存在 */
/* ❌ --error-1 到 --error-12 不存在 */
/* ❌ --warning-1 到 --warning-12 不存在 */
```

**用户影响**:
- 错误提示无背景色 → 无法识别
- 成功提示无背景色 → 用户不知道操作成功了
- WCAG 可访问性不符合

**修复难度**: ⭐ 低 (45 分钟)

---

### 3. Font Weight 非法值 (typography.ts)

**问题**:
```typescript
fontWeight: {
  bold: 'Bold',      // ❌ CSS 需要 700
  medium: 'Medium',  // ❌ CSS 需要 500
  regular: 'Regular' // ❌ CSS 需要 400
}
```

**用户影响**:
- 文本无适当权重，所有文本看起来权重相同
- 标题不够突出，内容看不清
- 排版层级混乱

**修复难度**: ⭐ 低 (15 分钟)

---

## 🟡 中等优先级问题

### 4. 样式分散 (styles.ts + Button.tsx + Header.tsx 等)

**问题**: Button 的样式定义在 4 个不同的地方

```
修改所有按钮高度：
1. styles.ts 中 btnPrimaryStyle
2. Button.tsx 中 sizeStyles.md
3. css.ts 中 .submit-btn-active
4. tokens/layout.ts 中 button size
5. 还有其他硬编码的 40px...

所以需要改 5+ 个地方 ❌
```

**修复难度**: ⭐⭐⭐ 高 (需要重构)

---

### 5. ARIA 缺失 (Header.tsx, Button.tsx 等)

**问题**:
```typescript
// Header.tsx New Chat 按钮
<button className={getIconBtnClass(...)} onClick={onNewChat}>
  <Plus size={12} />
  <span>新设计</span>
</button>

// ❌ 问题：
// - 使用 className 表达语义（应该用 aria-* 属性）
// - 屏幕阅读器只读到 "新设计"，不知道是按钮
// - 键盘用户可能无法访问（如果实现有问题）
```

**用户影响**: 屏幕阅读器用户无法正确理解界面

**修复难度**: ⭐⭐ 中 (2-3 小时)
